## 九、组合应用示例

### 口语版流水账
&gt; 周一跟产品吵了一架，搞了一下登录功能，弄了一下发现后端接口变了，催了一下后端，周二终于搞定了。周三整了一下支付页面，复制了一下之前的代码，网上找了个方案，调了一下参数。周四开了个会，说要做个新功能，还没想好怎么做。周五写了个文档，随便整理了一下，发群里了。

### 专业版转化
&gt; **周一**：与产品团队进行多轮深度对齐，推进用户认证模块开发，经排查确认为后端接口规范变更，已督促后端团队当日完成修正，顺利完成该功能交付。
&gt;
&gt; **周二-周三**：推进支付模块前端开发，复用既有订单页面架构，参考业界最佳实践实现支付流程，优化接口调用参数配置。
&gt;
&gt; **周四**：组织创新方向研讨会，探索智能推荐功能可行性，目前处于方案选型期。
&gt;
&gt; **周五**：输出支付模块操作手册，体系化梳理核心流程，已同步至项目组成员。

## 十、禁用词清单（避免过度包装）

以下词汇虽然"高大上"，但容易引起反感，**谨慎使用**：

| ❌ 过度包装词 | ✅ 建议替换为 |
|-------------|-------------|
| 赋能 | 支持/提供能力/优化工具 |
| 抓手 | 切入点/关键措施/具体方法 |
| 闭环 | 完成/解决/流程结束 |
| 底层逻辑 | 核心原理/根本原因/机制 |
| 顶层设计 | 整体规划/架构设计 |
| 颗粒度 | 细节程度/细化程度 |
| 组合拳 | 多项措施/组合方案 |
| 引爆点 | 关键因素/转折点 |
| 生态化反 | （避免使用，说人话） |
| 端到端 | 全流程/全链路 |

## 十一、动态替换规则

### 规则1：动词+名词强化
- 弱："做了代码优化" → 强："完成性能瓶颈治理"
- 弱："改了Bug" → 强："修复内存泄漏缺陷"

### 规则2：增加结果导向
- 弱："测了一下" → 强："验证通过，符合上线标准"
- 弱："沟通了一下" → 强："达成共识，确认排期"

### 规则3：量化具体化
- 弱："提升了很多" → 强："QPS从500提升至1200"
- 弱："花了不少时间" → 强："投入3人日"

### 规则4：去除情绪化
- 弱："后端不配合" → 强："存在跨部门协同gap"
- 弱："需求老是变" → 强："需求变更频率较高，建议强化评审"
